# SoC Test

SoC-level test integration is the discipline of orchestrating the many individual DFT structures, scan chains and compression for each core, memory BIST, logic BIST, JTAG/IEEE 1500 wrappers, analog/mixed-signal test, into a coherent chip-level test architecture that can be driven efficiently from the tester. The central element is the test access mechanism (TAM), the on-chip routing and multiplexing infrastructure that connects external test pins to internal test resources, typically organized hierarchically: a chip-level JTAG TAP controller provides the entry point, from which test instructions are decoded and routed to IEEE 1500-wrapped cores, MBIST controllers, and LBIST controllers, often through an internal test access bus or dedicated test network.

A key SoC test challenge is managing the sheer scale and heterogeneity of modern designs: a chip may contain dozens of cores from different IP providers, each delivered with its own pre-verified test patterns and DFT architecture, requiring pattern retargeting (translating core-level patterns to chip-level pin sequences through the wrapper and TAM hierarchy) rather than flat, full-chip ATPG. Test scheduling determines which tests run in parallel versus sequentially, balancing total test time against power constraints, since running many BIST engines or scan chains simultaneously can exceed the chip's power delivery or thermal limits during test (a phenomenon distinct from, but related to, functional power management).

Production test programs for SoCs typically run through multiple insertions: wafer sort (probe test before packaging, focused on identifying gross defects early and cheaply), final test (full pattern set including at-speed, BIST, and analog tests after packaging), and burn-in or system-level test (SLT) for high-reliability applications, where the device runs representative workloads to catch defects that structural patterns miss. Power-aware test is increasingly important: scan shift with many flops toggling simultaneously can draw far more current than functional operation, requiring techniques like low-power scan cells, shift clock gating, and pattern reordering to stay within IR-drop and thermal budgets. Coordinating fault coverage, test time, power, and pattern count across this entire hierarchy is the core job of SoC-level DFT engineers.
